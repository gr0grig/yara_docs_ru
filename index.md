# Добро пожаловать в документацию YARA!

YARA — это инструмент, предназначенный (но не ограниченный этим) для помощи исследователям вредоносного ПО в идентификации и классификации образцов вредоносных программ. С помощью YARA вы можете создавать описания семейств вредоносного ПО (или чего угодно, что вы хотите описать) на основе текстовых или бинарных паттернов. Каждое описание, также называемое правилом, состоит из набора строк и логического выражения, которое определяет его логику. Рассмотрим пример:

```yara
rule silent_banker : banker
{
    meta:
        description = "This is just an example"
        threat_level = 3
        in_the_wild = true
    strings:
        $a = {6A 40 68 00 30 00 00 6A 14 8D 91}
        $b = {8D 4D B0 2B C1 83 C0 27 99 6A 4E 59 F7 F9}
        $c = "UVODFRYSIHLNWPEJXQZAKCBGMT"
    condition:
        $a or $b or $c
}
```

Приведённое выше правило говорит YARA, что любой файл, содержащий одну из трёх строк, должен быть обозначен как silent_banker. Это лишь простой пример — более сложные и мощные правила можно создавать с использованием подстановочных символов, строк без учёта регистра, регулярных выражений, специальных операторов и множества других возможностей, которые описаны в данной документации.
