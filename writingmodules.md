# Написание собственных модулей

Впервые в YARA 3.0 появилась возможность расширять функциональность для выражения более сложных и точных условий. YARA 3.0 делает это с помощью модулей, которые позволяют определять структуры данных и функции, которые затем можно использовать в ваших правилах. Примеры того, что может делать модуль, вы можете увидеть в разделе [Использование модулей](writingrules.md#использование-модулей).

Цель последующих разделов -- научить вас создавать собственные модули, чтобы добавить в YARA ту замечательную функцию, о которой вы всегда мечтали.


## Модуль "Hello World!"

Модули пишутся на языке C и встраиваются в YARA в процессе компиляции. Для создания собственных модулей необходимо быть знакомым с языком программирования C и уметь конфигурировать и собирать YARA из исходного кода. Вам не нужно понимать, как YARA творит свою магию; YARA предоставляет простой API для модулей, и это все, что вам нужно знать.

Исходный код вашего модуля должен находиться в каталоге *libyara/modules* дерева исходного кода. Рекомендуется использовать имя модуля в качестве имени файла исходного кода; если ваш модуль называется *foo*, его файл исходного кода должен называться *foo.c*.

В каталоге *libyara/modules* вы найдете файл *demo.c*, который мы будем использовать в качестве отправной точки. Файл выглядит следующим образом:

```c
#include <yara/modules.h>

#define MODULE_NAME demo

begin_declarations;

  declare_string("greeting");

end_declarations;

int module_initialize(
    YR_MODULE* module)
{
  return ERROR_SUCCESS;
}

int module_finalize(
    YR_MODULE* module)
{
  return ERROR_SUCCESS;
}

int module_load(
    YR_SCAN_CONTEXT* context,
    YR_OBJECT* module_object,
    void* module_data,
    size_t module_data_size)
{
  yr_set_string("Hello World!", module_object, "greeting");
  return ERROR_SUCCESS;
}

int module_unload(
    YR_OBJECT* module_object)
{
  return ERROR_SUCCESS;
}

#undef MODULE_NAME
```

Давайте разберем исходный код, чтобы вы могли понять каждую деталь. Первая строка в коде:

```c
#include <yara/modules.h>
```

Заголовочный файл *modules.h* содержит определения API модулей YARA, поэтому эта директива включения обязательна во всех ваших модулях. Вторая строка:

```c
#define MODULE_NAME demo
```

Так определяется имя вашего модуля, и это тоже обязательно. Каждый модуль должен определить свое имя в начале исходного кода. Имена модулей должны быть уникальными среди всех модулей, встроенных в YARA.

Далее следует раздел объявлений:

```c
begin_declarations;

  declare_string("greeting");

end_declarations;
```

Именно здесь модуль объявляет функции и структуры данных, которые будут доступны для ваших правил YARA. В данном случае мы объявляем всего лишь строковую переменную с именем *greeting*. Мы обсудим эти концепции более подробно в разделе [Раздел объявлений](#раздел-объявлений).

После раздела объявлений вы найдете пару функций:

```c
int module_initialize(
    YR_MODULE* module)
{
  return ERROR_SUCCESS;
}

int module_finalize(
    YR_MODULE* module)
{
  return ERROR_SUCCESS;
}
```

Функция `module_initialize` вызывается при инициализации YARA, а её парная функция `module_finalize` вызывается при завершении работы YARA. Эти функции позволяют вам инициализировать и освобождать любые глобальные структуры данных, которые могут понадобиться вашему модулю.

Затем идет функция `module_load`:

```c
int module_load(
    YR_SCAN_CONTEXT* context,
    YR_OBJECT* module_object,
    void* module_data,
    size_t module_data_size)
{
  set_string("Hello World!", module_object, "greeting");
  return ERROR_SUCCESS;
}
```

Эта функция вызывается один раз для каждого сканируемого файла, но только если модуль импортирован каким-либо правилом с помощью директивы `import`. Функция `module_load` -- это место, где ваш модуль имеет возможность инспектировать сканируемый файл, разбирать или анализировать его предпочтительным способом, а затем заполнять структуры данных, определенные в разделе объявлений.

В этом примере функция `module_load` вообще не инспектирует содержимое файла, она просто присваивает строку "Hello World!" ранее объявленной переменной *greeting*.

И наконец, у нас есть функция `module_unload`:

```c
int module_unload(
    YR_OBJECT* module_object)
{
  return ERROR_SUCCESS;
}
```

Для каждого вызова `module_load` есть соответствующий вызов `module_unload`. Эта функция позволяет вашему модулю освободить любые ресурсы, выделенные во время `module_load`. В данном случае нечего освобождать, поэтому функция просто возвращает `ERROR_SUCCESS`. Обе функции `module_load` и `module_unload` должны возвращать `ERROR_SUCCESS`, чтобы указать, что все прошло нормально. Если возвращается другое значение, сканирование будет прервано и пользователю будет сообщено об ошибке.

### Сборка нашего "Hello World!"

Модули не встраиваются в YARA волшебным образом просто при помещении их исходного кода в каталог *libyara/modules*; необходимо выполнить еще два шага, чтобы они заработали. Первый шаг -- добавление вашего модуля в файл *module_list*, который также находится в каталоге *libyara/modules*.

Файл *module_list* выглядит так:

```
MODULE(tests)
MODULE(pe)

#ifdef CUCKOO_MODULE
MODULE(cuckoo)
#endif
```

Вы должны добавить строку *MODULE(<имя>)* с именем вашего модуля в этот файл. В нашем случае итоговый *module_list* будет таким:

```
MODULE(tests)
MODULE(pe)

#ifdef CUCKOO_MODULE
MODULE(cuckoo)
#endif

MODULE(demo)
```

Второй шаг -- модификация файла *Makefile.am*, чтобы сообщить программе *make*, что исходный код вашего модуля должен быть скомпилирован и слинкован с YARA. В самом начале *Makefile.am* вы найдете следующее:

```
MODULES =  libyara/modules/tests/tests.c
MODULES += libyara/modules/pe/pe.c

if CUCKOO_MODULE
MODULES += libyara/modules/cuckoo/cuckoo.c
endif
```

Просто добавьте новую строку для вашего модуля:

```
MODULES =  libyara/modules/tests/tests.c
MODULES += libyara/modules/pe/pe.c

if CUCKOO_MODULE
MODULES += libyara/modules/cuckoo/cuckoo.c
endif

MODULES += libyara/modules/demo/demo.c
```

И это все! Теперь вы готовы собрать YARA с вашим совершенно новым модулем. Просто перейдите в корневой каталог дерева исходного кода и введите как обычно:

```
./bootstrap.sh
./configure
make
sudo make install
```

Теперь вы сможете создать правило вроде этого:

```yara
import "demo"

rule HelloWorld
{
    condition:
        demo.greeting == "Hello World!"
}
```

Любой файл, просканированный с этим правилом, будет ему соответствовать, потому что `demo.greeting == "Hello World!"` всегда истинно.

## Раздел объявлений

Раздел объявлений -- это место, где вы объявляете переменные, структуры и функции, которые будут доступны для ваших правил YARA. Каждый модуль должен содержать раздел объявлений следующего вида:

```
begin_declarations;

    <ваши объявления здесь>

end_declarations;
```

### Базовые типы

В разделе объявлений вы можете использовать `declare_string(<имя переменной>)`, `declare_integer(<имя переменной>)` и `declare_float(<имя переменной>)` для объявления строковых, целочисленных или вещественных переменных соответственно. Например:

```
begin_declarations;

    declare_integer("foo");
    declare_string("bar");
    declare_float("baz");

end_declarations;
```

> **Примечание:** Переменные с плавающей точкой требуют YARA версии 3.3.0 или выше.

Имена переменных могут содержать только буквы, цифры и символы подчеркивания. Эти переменные затем могут использоваться в ваших правилах везде, где ожидается целое число или строка. Предполагая, что ваш модуль называется "mymodule", их можно использовать так:

```
mymodule.foo > 5

mymodule.bar matches /someregexp/
```


### Структуры

Ваши объявления можно организовать более структурированным образом:

```
begin_declarations;

    declare_integer("foo");
    declare_string("bar");
    declare_float("baz");

    begin_struct("some_structure");

        declare_integer("foo");

        begin_struct("nested_structure");

            declare_integer("bar");

        end_struct("nested_structure");

    end_struct("some_structure");

    begin_struct("another_structure");

        declare_integer("foo");
        declare_string("bar");
        declare_string("baz");
        declare_float("tux");

    end_struct("another_structure");

end_declarations;
```

В этом примере мы используем `begin_struct(<имя структуры>)` и `end_struct(<имя структуры>)` для определения двух структур с именами *some_structure* и *another_structure*. Внутри ограничителей структуры вы можете размещать любые другие объявления, включая объявления вложенных структур. Также обратите внимание, что члены различных структур могут иметь одинаковые имена, но члены внутри одной структуры должны иметь уникальные имена.

При обращении к этим переменным из ваших правил это будет выглядеть так:

```
mymodule.foo
mymodule.some_structure.foo
mymodule.some_structure.nested_structure.bar
mymodule.another_structure.baz
```


### Массивы

Так же, как вы объявляете отдельные строки, целые числа, вещественные числа или структуры, вы можете объявлять их массивы:

```
begin_declarations;

    declare_integer_array("foo");
    declare_string_array("bar");
    declare_float_array("baz");

    begin_struct_array("struct_array");

        declare_integer("foo");
        declare_string("bar");

    end_struct_array("struct_array");

end_declarations;
```

Отдельные значения в массиве используются так же, как в большинстве языков программирования:

```
foo[0]
bar[1]
baz[3]
struct_array[4].foo
struct_array[1].bar
```

Массивы имеют нулевую индексацию и не имеют фиксированного размера; они растут по мере необходимости, когда вы начинаете инициализировать их значения.


### Словари

*Добавлено в версии 3.2.0*

Вы также можете объявлять словари целых чисел, вещественных чисел, строк или структур:

```
begin_declarations;

    declare_integer_dictionary("foo");
    declare_string_dictionary("bar");
    declare_float_dictionary("baz")

    begin_struct_dictionary("struct_dict");

        declare_integer("foo");
        declare_string("bar");

    end_struct_dictionary("struct_dict");

end_declarations;
```

Доступ к отдельным значениям в словаре осуществляется с помощью строкового ключа:

```
foo["somekey"]
bar["anotherkey"]
baz["yetanotherkey"]
struct_dict["k1"].foo
struct_dict["k1"].bar
```

### Функции

Одна из наиболее мощных возможностей модулей YARA -- возможность объявления функций, которые затем можно вызывать из ваших правил. Функции должны объявляться в разделе объявлений следующим образом:

```
declare_function(<имя функции>, <типы аргументов>, <тип возвращаемого значения>, <функция C>);
```

*<имя функции>* -- это имя, которое будет использоваться в ваших правилах YARA для вызова функции.

*<типы аргументов>* -- это строка, содержащая по одному символу на каждый аргумент функции, где символ указывает тип аргумента. Функции могут принимать четыре различных типа аргументов: строка, целое число, вещественное число и регулярное выражение, обозначаемые символами: **s**, **i**, **f** и **r** соответственно. Если ваша функция принимает два целых числа, *<типы аргументов>* должно быть *"ii"*, если она принимает целое число в качестве первого аргумента и строку в качестве второго -- *<типы аргументов>* должно быть *"is"*, если она принимает три строки и вещественное число -- *<типы аргументов>* должно быть *"sssf"*.

*<тип возвращаемого значения>* -- это строка с одним символом, указывающим тип возвращаемого значения. Возможные типы возвращаемых значений: строка (*"s"*), целое число (*"i"*) и вещественное число (*"f"*).

*<функция C>* -- это идентификатор фактической реализации вашей функции.

Вот полный пример:

```c
define_function(isum)
{
  int64_t a = integer_argument(1);
  int64_t b = integer_argument(2);

  return_integer(a + b);
}

define_function(fsum)
{
  double a = float_argument(1);
  double b = float_argument(2);

  return_integer(a + b);
}

begin_declarations;

    declare_function("sum", "ii", "i", sum);

end_declarations;
```

Как видно из приведенного выше примера, код вашей функции должен быть определен перед разделом объявлений, вот так:

```
define_function(<идентификатор функции>)
{
  ...ваш код здесь
}
```

Функции могут быть перегружены, как в C++ и других языках программирования. Вы можете объявить две функции с одинаковым именем, если они отличаются типом или количеством аргументов. Пример перегруженных функций можно найти в [модуле Hash](modules/hash.md); в нем есть две функции для вычисления хешей MD5: одна принимает смещение и длину внутри файла, а другая принимает строку:

```
begin_declarations;

    declare_function("md5", "ii", "s", data_md5);
    declare_function("md5", "s", "s", string_md5);

end_declarations;
```

Мы обсудим реализацию функций более подробно в разделе [Подробнее о функциях](#подробнее-о-функциях).

## Инициализация и завершение

Каждый модуль должен реализовать две функции для инициализации и завершения: `module_initialize` и `module_finalize`. Первая вызывается при инициализации YARA функцией `yr_initialize`, а вторая -- при завершении функцией `yr_finalize`. Обе функции вызываются независимо от того, импортируется ли модуль каким-либо правилом или нет.

Эти функции дают вашему модулю возможность инициализировать любые глобальные структуры данных, которые ему могут понадобиться, но в большинстве случаев они являются просто пустыми функциями:

```c
int module_initialize(
    YR_MODULE* module)
{
  return ERROR_SUCCESS;
}

int module_finalize(
    YR_MODULE* module)
{
  return ERROR_SUCCESS;
}
```

Любое возвращаемое значение, отличное от `ERROR_SUCCESS`, приведет к прерыванию выполнения YARA.

## Реализация логики модуля

Помимо `module_initialize` и `module_finalize`, каждый модуль должен реализовать еще две функции, которые вызываются YARA при сканировании файла или области памяти процесса: `module_load` и `module_unload`. Обе функции вызываются один раз для каждого сканируемого файла или процесса, но только если модуль был импортирован с помощью директивы `import`. Если модуль не импортирован никаким правилом, ни `module_load`, ни `module_unload` не будут вызваны.

Функция `module_load` имеет следующий прототип:

```c
int module_load(
    YR_SCAN_CONTEXT* context,
    YR_OBJECT* module_object,
    void* module_data,
    size_t module_data_size)
```

Аргумент `context` содержит информацию, относящуюся к текущему сканированию, включая сканируемые данные. Аргумент `module_object` -- это указатель на структуру `YR_OBJECT`, связанную с модулем. Каждая структура, переменная или функция, объявленная в модуле YARA, представлена структурой `YR_OBJECT`. Эти структуры образуют дерево, корнем которого является структура `YR_OBJECT` модуля. Если у вас есть следующие объявления в модуле с именем *mymodule*:

```
begin_declarations;

    declare_integer("foo");

    begin_struct("bar");

        declare_string("baz");

    end_struct("bar");

end_declarations;
```

То дерево будет выглядеть так:

```
 YR_OBJECT(type=OBJECT_TYPE_STRUCT, name="mymodule")
  |
  |_ YR_OBJECT(type=OBJECT_TYPE_INTEGER, name="foo")
  |
  |_ YR_OBJECT(type=OBJECT_TYPE_STRUCT, name="bar")
      |
      |_ YR_OBJECT(type=OBJECT_TYPE_STRING, name="baz")
```

Обратите внимание, что и *bar*, и *mymodule* имеют один и тот же тип `OBJECT_TYPE_STRUCT`, что означает, что `YR_OBJECT`, связанный с модулем, -- это просто еще одна структура, такая же как *bar*. Фактически, когда вы пишете в своих правилах что-то вроде `mymodule.foo`, вы выполняете поиск поля в структуре точно так же, как это делает `bar.baz`.

Таким образом, аргумент `module_object` позволяет вам получить доступ к каждой переменной, структуре или функции, объявленной модулем, предоставляя указатель на корень дерева объектов.

Аргумент `module_data` -- это указатель на любые дополнительные данные, передаваемые модулю, а `module_data_size` -- размер этих данных. Не всем модулям нужны дополнительные данные; большинство из них полагаются исключительно на сканируемые данные, но некоторым требуется дополнительная информация на входе. Хороший пример -- [модуль Cuckoo](modules/cuckoo.md), который получает отчет о поведении, связанный со сканируемыми PE-файлами; этот отчет передается через аргументы `module_data` и `module_data_size`.

Для получения дополнительной информации о том, как передавать дополнительные данные вашему модулю, ознакомьтесь с аргументом `-x` в разделе [Командная строка](commandline.md).

### Доступ к сканируемым данным

Большинству модулей YARA необходим доступ к файлу или памяти процесса, которые сканируются, для извлечения из них информации. Сканируемые данные передаются модулю в структуре `YR_SCAN_CONTEXT`, передаваемой в функцию `module_load`. Данные иногда разделены на блоки, поэтому ваш модуль должен итерировать по блокам с помощью макроса `foreach_memory_block`:

```c
int module_load(
    YR_SCAN_CONTEXT* context,
    YR_OBJECT* module_object,
    void* module_data,
    size_t module_data_size)
{
    YR_MEMORY_BLOCK* block;

    foreach_memory_block(context, block)
    {
        ..сделать что-то с текущим блоком памяти
    }
}
```

Каждый блок памяти представлен структурой `YR_MEMORY_BLOCK` со следующими атрибутами:

**`YR_MEMORY_BLOCK_FETCH_DATA_FUNC fetch_data`**

Указатель на функцию, возвращающую указатель на данные блока.

**`size_t size`**

Размер блока данных.

**`size_t base`**

Базовое смещение/адрес для этого блока. Если сканируется файл, это поле содержит смещение внутри файла, где начинается блок; если сканируется адресное пространство процесса, это поле содержит виртуальный адрес, с которого начинается блок.

Блоки всегда итерируются в том же порядке, в котором они появляются в файле или памяти процесса. В случае файлов первый блок будет содержать начало файла. Как правило, один блок будет содержать все содержимое файла в большинстве случаев, но вы не можете полагаться на это при написании кода. Для очень больших файлов YARA может разделить файл на два или более блоков, и ваш модуль должен быть готов к этому.

Ситуация совершенно иная для процессов. При сканировании адресного пространства процесса ваш модуль определенно получит большое количество блоков -- по одному для каждого выделенного региона памяти в адресном пространстве процесса.

Однако в некоторых случаях вам не нужно итерировать по блокам. Если ваш модуль просто разбирает заголовок какого-либо формата файла, вы можете спокойно предположить, что весь заголовок содержится в первом блоке (тем не менее, добавьте проверки в свой код). В таких случаях вы можете использовать макрос `first_memory_block`:

```c
int module_load(
    YR_SCAN_CONTEXT* context,
    YR_OBJECT* module_object,
    void* module_data,
    size_t module_data_size)
{
    YR_MEMORY_BLOCK* block;
    const uint8_t* block_data;

    block = first_memory_block(context);
    if (block != NULL)
    {
      block_data = yr_fetch_block_data(block)

      if (block_data != NULL)
      {
        ..сделать что-то с блоком памяти
      }
    }
}
```

В предыдущем примере вы также можете увидеть, как использовать функцию `yr_fetch_block_data`. Эта функция принимает указатель на сам блок (как указатель `self` или `this`) и возвращает указатель на данные блока. Ваш модуль не является владельцем памяти, на которую указывает этот указатель; освобождение этой памяти -- не ваша ответственность. Однако имейте в виду, что указатель действителен только до тех пор, пока вы не запросите следующий блок памяти. До тех пор, пока вы используете указатель в пределах области видимости `foreach_memory_block`, все в порядке. Также учитывайте, что `yr_fetch_block_data` может вернуть указатель NULL; ваш код должен быть готов к этому случаю.

```c
const uint8_t* block_data;

foreach_memory_block(context, block)
{
  block_data = yr_fetch_block_data(block);

  if (block_data != NULL)
  {
    // использование block_data здесь безопасно.
  }
}

// память, на которую указывает block_data, может быть уже освобождена здесь.
```


### Установка значений переменных

Функция `module_load` -- это место, где вы присваиваете значения переменным, объявленным в разделе объявлений, после того как вы разобрали или проанализировали сканируемые данные и/или любые дополнительные данные модуля. Это делается с помощью функций `set_float`, `set_integer` и `set_string`:

**`void set_float(double value, YR_OBJECT* object, const char* field, ...)`**

**`void set_integer(int64_t value, YR_OBJECT* object, const char* field, ...)`**

**`void set_string(const char* value, YR_OBJECT* object, const char* field, ...)`**

Эти функции принимают значение, которое нужно присвоить переменной, указатель на `YR_OBJECT`, представляющий саму переменную или какого-либо предка этой переменной, дескриптор поля и дополнительные аргументы, определяемые дескриптором поля.

Если мы присваиваем значение переменной, представленной самим `object`, то дескриптор поля должен быть `NULL`. Например, предполагая, что `object` указывает на структуру `YR_OBJECT`, соответствующую какой-либо целочисленной переменной, мы можем установить значение этой целочисленной переменной так:

```c
set_integer(<значение>, object, NULL);
```

Дескриптор поля используется, когда вы хотите присвоить значение какому-либо потомку `object`. Например, рассмотрим следующие объявления:

```
begin_declarations;

    begin_struct("foo");

        declare_string("bar");

        begin_struct("baz");

            declare_integer("qux");

        end_struct("baz");

    end_struct("foo");

end_declarations;
```

Если `object` указывает на `YR_OBJECT`, связанный со структурой `foo`, вы можете установить значение строки `bar` так:

```c
set_string(<значение>, object, "bar");
```

А значение `qux` так:

```c
set_integer(<значение>, object, "baz.qux");
```

Помните, что аргумент `module_object` функции `module_load` был указателем на `YR_OBJECT`? Помните, что этот `YR_OBJECT` -- это структура, такая же как `foo`? Ну так вот, вы также можете установить значения `bar` и `qux` вот так:

```c
set_string(<значение>, module_object, "foo.bar");
set_integer(<значение>, module_object, "foo.baz.qux");
```

Но что с массивами? Как мне установить значение элементов массива? Если у вас есть следующие объявления:

```
begin_declarations;

    declare_integer_array("foo");

    begin_struct_array("bar")

        declare_string("baz");
        declare_integer_array("qux");

    end_struct_array("bar");

end_declarations;
```

То все следующие выражения допустимы:

```c
set_integer(<значение>, module, "foo[0]");
set_integer(<значение>, module, "foo[%i]", 2);
set_string(<значение>, module, "bar[%i].baz", 5);
set_string(<значение>, module, "bar[0].qux[0]");
set_string(<значение>, module, "bar[0].qux[%i]", 0);
set_string(<значение>, module, "bar[%i].qux[%i]", 100, 200);
```

Вхождения `%i` в дескрипторе поля заменяются дополнительными целочисленными аргументами, переданными в функцию. Это работает так же, как `printf` в программах на C, но допускаются только спецификаторы формата `%i` и `%s` -- для целочисленных и строковых аргументов соответственно.

Спецификатор формата `%s` используется для присвоения значений определенному ключу в словаре:

```c
set_integer(<значение>, module, "foo[\"key\"]");
set_integer(<значение>, module, "foo[%s]", "key");
set_string(<значение>, module, "bar[%s].baz", "another_key");
```

Если вы явно не присвоите значение объявленной переменной, элементу массива или словаря, оно останется в неопределенном состоянии. Это не проблема и даже полезно во многих случаях. Например, если ваш модуль разбирает файлы определенного формата и получает файл другого формата, вы можете спокойно оставить все свои переменные неопределенными, вместо того чтобы присваивать им бессмысленные значения. YARA обрабатывает неопределенные значения в условиях правил, как описано в разделе [Использование модулей](writingrules.md#использование-модулей).

Помимо функций `set_float`, `set_integer` и `set_string`, у вас есть их парные функции `get_float`, `get_integer` и `get_string`. Как следует из названий, они используются для получения значения переменной, что может быть полезно при реализации ваших функций для извлечения значений, ранее сохраненных функцией `module_load`.

**`double get_float(YR_OBJECT* object, const char* field, ...)`**

**`int64_t get_integer(YR_OBJECT* object, const char* field, ...)`**

**`SIZED_STRING* get_string(YR_OBJECT* object, const char* field, ...)`**

Также существует функция для получения любого `YR_OBJECT` в дереве объектов:

**`YR_OBJECT* get_object(YR_OBJECT* object, const char* field, ...)`**

А теперь небольшой экзамен...

Эквивалентны ли следующие две строки? Почему?

```c
set_integer(1, get_object(module_object, "foo.bar"), NULL);
set_integer(1, module_object, "foo.bar");
```

### Сохранение данных для последующего использования

Иногда информации, хранящейся непосредственно в ваших переменных с помощью `set_integer` и `set_string`, недостаточно. Вам может понадобиться хранить более сложные структуры данных или информацию, которую не нужно предоставлять правилам YARA.

Хранение информации необходимо, когда ваш модуль экспортирует функции для использования в правилах YARA. Реализация этих функций обычно требует доступа к информации, сгенерированной `module_load`, которая должна где-то храниться. У вас может возникнуть соблазн определить глобальные переменные для хранения необходимой информации, но это сделает ваш код небезопасным для многопоточности. Правильный подход -- использование поля `data` структур `YR_OBJECT`.

Каждый `YR_OBJECT` имеет поле `void* data`, которое может безопасно использоваться вашим кодом для хранения указателя на любые необходимые данные. Типичный паттерн -- использование поля `data` `YR_OBJECT` модуля, как в следующем примере:

```c
typedef struct _MY_DATA
{
   int some_integer;

} MY_DATA;

int module_load(
    YR_SCAN_CONTEXT* context,
    YR_OBJECT* module_object,
    void* module_data,
    size_t module_data_size)
{
    module_object->data = yr_malloc(sizeof(MY_DATA));
    ((MY_DATA*) module_object->data)->some_integer = 0;

    return ERROR_SUCCESS;
}
```

Не забудьте освободить выделенную память в функции `module_unload`:

```c
int module_unload(
    YR_OBJECT* module_object)
{
    yr_free(module_object->data);

    return ERROR_SUCCESS;
}
```

> **Предупреждение:** Не используйте глобальные переменные для хранения данных. Функции в модуле могут вызываться из разных потоков одновременно, и это может привести к повреждению данных или некорректному поведению.

## Подробнее о функциях

Мы уже показали, как объявить функцию в разделе [Раздел объявлений](#функции). Здесь мы обсудим, как предоставить для них реализацию.

### Аргументы функций

Внутри кода функции вы получаете её аргументы с помощью `integer_argument(n)`, `float_argument(n)`, `regexp_argument(n)`, `string_argument(n)` или `sized_string_argument(n)` в зависимости от типа аргумента, где *n* -- номер аргумента, начиная с 1.

`string_argument(n)` можно использовать, когда ваша функция ожидает получить строку C, завершающуюся нулевым символом; если ваша функция может получать произвольные двоичные данные, возможно содержащие нулевые символы, вы должны использовать `sized_string_argument(n)`.

Вот несколько примеров:

```c
int64_t arg_1 = integer_argument(1);
RE* arg_2 = regexp_argument(2);
char* arg_3 = string_argument(3);
SIZED_STRING* arg_4 = sized_string_argument(4);
double arg_5 = float_argument(1);
```

Тип C для целочисленных аргументов -- `int64_t`, для вещественных аргументов -- `double`, для регулярных выражений -- `RE*`, для строк, завершающихся нулевым символом, -- `char*`, а для строк, которые могут содержать нулевые символы, -- `SIZED_STRING*`. Структуры `SIZED_STRING` имеют следующие атрибуты:

**`SIZED_STRING`**

- `length` -- длина строки.
- `c_string` -- `char*`, указывающий на содержимое строки.

### Возвращаемые значения

Функции могут возвращать три типа значений: строки, целые числа и вещественные числа. Вместо использования оператора `return` языка C вы должны использовать `return_string(x)`, `return_integer(x)` или `return_float(x)` для возврата из функции в зависимости от типа возвращаемого значения. Во всех случаях *x* -- это константа, переменная или выражение, вычисляемое в `char*`, `int64_t` или `double` соответственно.

Вы можете использовать `return_string(YR_UNDEFINED)`, `return_float(YR_UNDEFINED)` и `return_integer(YR_UNDEFINED)` для возврата неопределенных значений из функции. Это полезно во многих ситуациях, например, если переданные функции аргументы не имеют смысла, или если ваш модуль ожидает определенный формат файла, а сканируемый файл имеет другой формат, или в любом другом случае, когда ваша функция не может вернуть допустимое значение.

> **Предупреждение:** Не используйте оператор `return` языка C для возврата из функции. Возвращаемое значение будет интерпретировано как код ошибки.

### Доступ к объектам

При написании функции иногда необходимо получить доступ к значениям, ранее присвоенным переменным модуля, или к дополнительным данным, хранящимся в поле `data` структур `YR_OBJECT`, как обсуждалось ранее в разделе [Сохранение данных для последующего использования](#сохранение-данных-для-последующего-использования). Но для этого нам нужен способ получить доступ к соответствующему `YR_OBJECT`. Для этого существуют две функции: `yr_module()` и `yr_parent()`. Функция `yr_module()` возвращает указатель на `YR_OBJECT` верхнего уровня, соответствующий модулю, -- тот же самый, что передается в функцию `module_load`. Функция `yr_parent()` возвращает указатель на `YR_OBJECT`, соответствующий структуре, в которой содержится функция. Например, рассмотрим следующий фрагмент кода:

```c
define_function(f1)
{
    YR_OBJECT* module = yr_module();
    YR_OBJECT* parent = yr_parent();

    // parent == module;
}

define_function(f2)
{
    YR_OBJECT* module = yr_module();
    YR_OBJECT* parent = yr_parent();

    // parent != module;
}

begin_declarations;

    declare_function("f1", "i", "i", f1);

    begin_struct("foo");

        declare_function("f2", "i", "i", f2);

    end_struct("foo");

end_declarations;
```

В `f1` переменная `module` указывает на `YR_OBJECT` верхнего уровня, так же как и переменная `parent`, потому что родителем для `f1` является сам модуль. В `f2`, однако, переменная `parent` указывает на `YR_OBJECT`, соответствующий структуре `foo`, тогда как `module` указывает на `YR_OBJECT` верхнего уровня, как и прежде.

### Контекст сканирования

Из функции вы также можете получить доступ к структуре `YR_SCAN_CONTEXT`, обсуждавшейся ранее в разделе [Доступ к сканируемым данным](#доступ-к-сканируемым-данным). Это полезно для функций, которым необходимо инспектировать сканируемый файл или память процесса. Вот как получить указатель на структуру `YR_SCAN_CONTEXT`:

```c
YR_SCAN_CONTEXT* context = yr_scan_context();
```
