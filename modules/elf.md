# Модуль ELF

*Добавлен в версии 3.2.0*

Модуль ELF очень похож на [модуль PE](pe.md), но предназначен для файлов формата ELF. Этот модуль предоставляет доступ к большинству полей заголовка ELF. Рассмотрим примеры:

```yara
import "elf"

rule single_section
{
    condition:
        elf.number_of_sections == 1
}

rule elf_64
{
    condition:
        elf.machine == elf.EM_X86_64
}
```

## Справочник

### type

Целое число с одним из следующих значений:

| Константа | Описание |
|-----------|----------|
| `ET_NONE` | Нет типа файла |
| `ET_REL` | Перемещаемый файл |
| `ET_EXEC` | Исполняемый файл |
| `ET_DYN` | Разделяемый объектный файл |
| `ET_CORE` | Файл дампа памяти |

*Пример: elf.type == elf.ET_EXEC*

### machine

Целое число с одним из следующих значений:

`EM_NONE`, `EM_M32`, `EM_SPARC`, `EM_386`, `EM_68K`, `EM_88K`, `EM_860`, `EM_MIPS`, `EM_MIPS_RS3_LE`, `EM_PPC`, `EM_PPC64`, `EM_ARM`, `EM_X86_64`, `EM_AARCH64`

*Пример: elf.machine == elf.EM_X86_64*

### entry_point

Смещение точки входа в файле или виртуальный адрес, в зависимости от того, сканируется ли файл или память процесса. Эквивалентно устаревшему ключевому слову `entrypoint`.

### number_of_sections

Количество секций в ELF-файле.

### sections

Массив объектов секций с нулевой индексацией, по одному для каждой секции ELF-файла. Отдельные секции доступны через оператор `[]`. Каждый объект секции имеет следующие атрибуты:

#### name

Имя секции.

*Пример: elf.sections[3].name == ".bss"*

#### size

Размер секции в байтах. Если тип секции не SHT_NOBITS, секция занимает sh_size байтов в файле. Секция типа SHT_NOBITS может иметь ненулевой размер, но не занимает места в файле.

#### offset

Смещение от начала файла до первого байта секции. Секция типа SHT_NOBITS не занимает места в файле, а её смещение указывает на концептуальное расположение в файле.

#### type

Целое число с одним из следующих значений:

| Константа | Описание |
|-----------|----------|
| `SHT_NULL` | Помечает секцию как неактивную |
| `SHT_PROGBITS` | Данные, определённые программой |
| `SHT_SYMTAB` | Таблица символов |
| `SHT_STRTAB` | Таблица строк |
| `SHT_RELA` | Записи перемещений |
| `SHT_HASH` | Хеш-таблица символов |
| `SHT_DYNAMIC` | Информация для динамической компоновки |
| `SHT_NOTE` | Информация, помечающая файл определённым образом |
| `SHT_NOBITS` | Не занимает места в файле, аналогична SHT_PROGBITS |
| `SHT_REL` | Записи перемещений |
| `SHT_SHLIB` | Зарезервирован, семантика не определена |
| `SHT_DYNSYM` | Символы динамической компоновки |

#### flags

Целое число с флагами секции:

| Константа | Описание |
|-----------|----------|
| `SHF_WRITE` | Секция содержит данные, доступные для записи |
| `SHF_ALLOC` | Секция занимает память при выполнении процесса |
| `SHF_EXECINSTR` | Секция содержит исполняемые машинные инструкции |

*Пример: elf.sections[2].flags & elf.SHF_WRITE*

#### address

*Добавлен в версии 3.6.0*

Виртуальный адрес, с которого начинается секция.

### number_of_segments

*Добавлен в версии 3.4.0*

Количество сегментов в ELF-файле.

### segments

*Добавлен в версии 3.4.0*

Массив объектов сегментов с нулевой индексацией. Каждый объект сегмента имеет следующие атрибуты:

#### alignment

Значение, определяющее выравнивание сегментов в памяти и в файле.

#### file_size

Количество байтов в образе файла для данного сегмента. Может быть равно нулю.

#### flags

Комбинация следующих флагов:

| Константа | Описание |
|-----------|----------|
| `PF_R` | Сегмент доступен для чтения |
| `PF_W` | Сегмент доступен для записи |
| `PF_X` | Сегмент исполняемый |

#### memory_size

Размер сегмента в памяти.

#### offset

Смещение от начала файла, где находится сегмент.

#### physical_address

Физический адрес сегмента (для систем, где это актуально).

#### type

Тип сегмента, один из: `PT_NULL`, `PT_LOAD`, `PT_DYNAMIC`, `PT_INTERP`, `PT_NOTE`, `PT_SHLIB`, `PT_PHDR`, `PT_LOPROC`, `PT_HIPROC`, `PT_GNU_STACK`.

#### virtual_address

Виртуальный адрес, по которому сегмент располагается в памяти.

### dynamic_section_entries

*Добавлен в версии 3.6.0*

Количество записей в динамической секции ELF-файла.

### dynamic

*Добавлен в версии 3.6.0*

Массив динамических объектов с нулевой индексацией. Каждый объект имеет следующие атрибуты:

#### type

Тип динамической секции. Встроенные значения: `DT_NULL`, `DT_NEEDED`, `DT_PLTRELSZ`, `DT_PLTGOT`, `DT_HASH`, `DT_STRTAB`, `DT_SYMTAB`, `DT_RELA`, `DT_RELASZ`, `DT_RELAENT`, `DT_STRSZ`, `DT_SYMENT`, `DT_INIT`, `DT_FINI`, `DT_SONAME`, `DT_RPATH`, `DT_SYMBOLIC`, `DT_REL`, `DT_RELSZ`, `DT_RELENT`, `DT_PLTREL`, `DT_DEBUG`, `DT_TEXTREL`, `DT_JMPREL`, `DT_BIND_NOW`, `DT_INIT_ARRAY`, `DT_FINI_ARRAY`, `DT_INIT_ARRAYSZ`, `DT_FINI_ARRAYSZ`, `DT_RUNPATH`, `DT_FLAGS`, `DT_ENCODING`.

#### value

Значение, связанное с данным типом. Тип значения (адрес, размер и т.д.) зависит от типа динамической записи.

### symtab_entries

*Добавлен в версии 3.6.0*

Количество записей в таблице символов ELF-файла.

### symtab

*Добавлен в версии 3.6.0*

Массив объектов символов с нулевой индексацией. Каждый объект имеет следующие атрибуты:

#### name

Имя символа.

#### value

Значение, связанное с символом. Обычно виртуальный адрес.

#### size

Размер символа.

#### type

Тип символа. Встроенные значения: `STT_NOTYPE`, `STT_OBJECT`, `STT_FUNC`, `STT_SECTION`, `STT_FILE`, `STT_COMMON`, `STT_TLS`.

#### bind

Привязка символа. Встроенные значения: `STB_LOCAL`, `STB_GLOBAL`, `STB_WEAK`.

#### shndx

Индекс секции, с которой связан символ.

### telfhash()

Функция, возвращающая Telfhash — TLSH-хеш экспортируемых и импортируемых символов ELF.

*Пример: elf.telfhash() == "t166a00284751084526486df8b5df5b2fccb3f511dbc188c37156f5e714a11bc5d71014d"*

### import_md5()

Функция, возвращающая Import Hash — MD5-хеш импортируемых символов ELF.

*Пример: elf.import_md5() == "c3eca50cbb03400a6e91b9fe48da0c0c"*
